# MVP-1 Acceptance

## Обязательные сценарии

1. Регистрация и логин клиента.
2. Поиск тура и просмотр карточки.
3. Создание заявки клиентом.
4. Обработка заявки менеджером до `ProposalSent`.
5. Подтверждение и закрытие заявки.

## Технические критерии

- gRPC-вызов `Booking -> Catalog` стабильно работает.
- Ошибки API отдаются в согласованном формате.
- Критичные переходы статусов покрыты тестами.

## Definition of Done

- User flows проходят вручную и через smoke.
- Unit + integration тесты зелёные.
- Нет блокеров уровня Critical/High.
